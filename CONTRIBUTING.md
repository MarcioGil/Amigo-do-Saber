# Guia de Contribuição

Obrigado por contribuir com o Amigo do Saber!

## Como contribuir
- Faça fork do repositório
- Crie uma branch para sua feature ou correção
- Envie um Pull Request detalhado
- Siga o padrão de código (ESLint/Prettier)
- Adicione testes para novas funcionalidades

## Padrões de código
- Use ESLint e Prettier
- Escreva funções pequenas e bem documentadas
- Prefira nomes descritivos

## Fluxo de Pull Request
- Descreva claramente o que foi alterado
- Relacione issues relevantes
- Adicione exemplos de uso/testes

## Comunicação
- Seja respeitoso e colaborativo
- Use o idioma português ou inglês

## Segurança
- Nunca exponha segredos em código ou PRs
- Use Key Vault/GitHub Secrets

## Testes
- Execute `npm test` no backend
- Execute `npx cypress run` no frontend

## Documentação
- Atualize o README se necessário
